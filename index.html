Glosario del Modulo 1

GitHub

GitHub es una plataforma de desarrollo colaborativo basada en la web que permite a los desarrolladores gestionar proyectos y controlar versiones de código fuente utilizando Git.

GitHub permite la colaboración en proyectos de software mediante un sistema de control de versiones, gestionando repositorios que permiten a los desarrolladores trabajar juntos en el mismo proyecto. Es muy usado en equipos de desarrollo ágil y proyectos de código abierto.

Buenas Prácticas de Programación

Las buenas prácticas en programación incluyen escribir código limpio, utilizar nombres descriptivos para las variables, documentar el código adecuadamente y seguir convenciones de estilo.

Las buenas prácticas garantizan que el código sea fácil de mantener, escalable y comprensible por otros desarrolladores. Involucra también hacer pruebas unitarias, revisar el código de otros (code reviews) y seguir patrones de diseño.

HTML y CSS

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. Define la estructura de la web. CSS (Cascading Style Sheets) es utilizado para dar formato y diseño a las páginas web, controlando colores, fuentes, y disposición.

HTML forma la estructura básica de las páginas web, mientras que CSS se utiliza para aplicar estilo y diseño visual a esos elementos. Juntos, permiten crear páginas web interactivas y atractivas. CSS también incluye características avanzadas como animaciones, flexbox y grid layout.

Inteligencia Artificial (IA)

La inteligencia artificial es una rama de la informática que se enfoca en la creación de sistemas capaces de realizar tareas que requieren inteligencia humana.

Los avances en IA incluyen el aprendizaje automático, procesamiento de lenguaje natural, visión computarizada, y la toma de decisiones autónomas. La IA está revolucionando muchos sectores, desde la medicina hasta la automoción.

Conceptos Git y GitHub

  • Git: Es un sistema de control de versiones que permite gestionar y rastrear los cambios realizados en el código fuente.
  • Ramas (Branches): Son versiones separadas del código.
  • git clone: Comando para copiar un repositorio remoto a tu computadora local.
  • git commit: Comando para guardar cambios realizados en el repositorio local.
  • git push: Comando para enviar cambios desde el repositorio local al repositorio remoto.
  • git fetch: Comando para obtener los cambios de un repositorio remoto, pero sin fusionarlos con tu rama actual. Esto te permite ver los cambios sin aplicarlos directamente.
  • origin: Es el nombre por defecto del repositorio remoto. Generalmente se refiere al repositorio desde el que clonaste tu proyecto, y es el repositorio al que envías tus cambios con git push.
  • git merge: Comando que se utiliza para combinar cambios de diferentes ramas dentro de tu repositorio. Se puede usar para fusionar la rama principal con una rama de características o viceversa.
  • Mezcla de código de otro repositorio: Puedes mezclar o fusionar el código de otro repositorio en el tuyo mediante un git fetch para traer los cambios y luego un git merge para integrarlos a tu rama local.

Git es un sistema distribuido que facilita el trabajo colaborativo en proyectos de software, permitiendo a los desarrolladores trabajar en diferentes ramas y fusionar cambios sin conflictos. GitHub agrega una capa de colaboración en la web, permitiendo revisar el código, crear pull requests, y gestionar proyectos.

git fetch es útil cuando deseas actualizar tu repositorio local con los cambios más recientes de un repositorio remoto, pero sin aplicar esos cambios directamente a tu rama activa. Esto te da la oportunidad de revisar los cambios antes de incorporarlos.

origin es el nombre predeterminado para el repositorio remoto. Cuando clonas un repositorio, Git asocia el nombre origin al repositorio desde el cual lo clonaste. Puedes usar este nombre para interactuar con el repositorio remoto en comandos como git push origin master o git pull origin feature-branch.

git merge permite combinar los cambios de una rama diferente con la rama activa. Este comando es esencial cuando trabajas en equipos, ya que permite que los cambios de diversas ramas se integren sin sobrescribir el trabajo de otros desarrolladores.

Mezclar código de otro repositorio implica incorporar cambios desde un repositorio remoto que no sea el tuyo. Esto es común cuando un proyecto utiliza código de otros proyectos o cuando deseas colaborar con repositorios externos.
